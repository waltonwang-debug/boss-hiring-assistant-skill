# 站内沟通服务

## 目标

只负责 BOSS 站内聊天推进，例如：

- 索要附件简历
- 确认候选人意向
- 收集可约时间
- 发送标准 FAQ 回复

不负责：

- 全量筛选
- 自动创建飞书日程

## 进入条件

只有在下面任一条件满足时才进入：

1. 用户明确要求“继续沟通这些候选人”
2. 每小时汇报中，用户明确选择了可推进沟通的候选人

## 消息发送规则

所有 BOSS 站内消息发送前，必须先读取：

- [boss-send-recipe.md](./boss-send-recipe.md)

并严格按其中的唯一主路径执行。

## 强约束动作

以下动作都属于强约束动作：

- 打开候选人详情
- 切换聊天线程
- 打开聊天线程
- 发送标准消息
- 索要附件简历
- 收集候选人可约时间
- 确认面试时间
- 发送飞书链接

这些动作必须：

1. 先准备最终 payload
2. 使用 `web-access` 主线程快路径
3. 做一次短时成功确认
4. 失败就进入可恢复暂停态

不允许：

- 长时间探索
- 连续试很多 DOM 变体
- 起子代理研究前端框架

## 默认消息策略

默认优先使用固定模板。
只有低风险 FAQ 且没有固定模板时，才允许受控生成简短回复。
