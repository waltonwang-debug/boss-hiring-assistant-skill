# 首次接入流程

## 目标

让用户只用一句自然语言请求，就能把这个 skill 接入龙虾，并进入正确的服务模式。

## 首次接入顺序

1. 先向用户介绍三类独立服务
2. 推荐用户先从 `boss-screening` 开始
3. 询问或判断用户本轮要启动哪一种服务
4. 按该服务引导对应的强制前置条件
5. 前置条件完成后，再开始自动读取当前账号下的在线岗位和 JD

## 首次接入时必须说明的三类服务

1. `boss-screening`
用于简历筛选、分类和周期汇报。
推荐首次先从这里开始。
需要顺带告诉用户：

- 可筛选的范围包括主动进入聊天列表的候选人、推荐页面候选人、以及具备搜索权益时的搜索结果
- 默认会按固定时间节奏自动扫描和汇报
- 用户可以覆盖默认时间节奏

2. `boss-chat`
用于 BOSS 站内沟通，例如索要附件简历、确认意向、收集时间。
只有在用户明确要求沟通候选人时才启动。

3. `boss-scheduling`
用于候选人已确认时间后的飞书约面。
只有在用户明确要求约面时才启动。

## 服务选择后的前置条件引导

### 如果用户选择 `boss-screening`

只引导这些前置条件：

1. 禁用或移除 `browser-use` 等浏览器 skill
2. 安装 `web-access`
   - GitHub 链接：`https://github.com/eze-is/web-access`
3. 切换到 `web-access`
4. 验证 `web-access` 已接管
5. 确认使用 Chrome / Chromium
6. 确认用户已登录 BOSS 网页端

不要提前引导飞书 bot。

同时必须向用户说明：

1. 本服务默认可筛选三类来源：
   - 主动聊天/主动投递
   - 推荐页面
   - 搜索页面（若账号有搜索权益）
2. 默认节奏是：
   - 周一到周五
   - 09:00-17:00
   - 每 60 分钟扫描一次
   - 每 60 分钟汇报一次
3. 用户若有自己的节奏，可以立即覆盖默认值

### 如果用户选择 `boss-chat`

引导：

1. 完成 `boss-screening` 同样的通用前置条件
2. 再确认本轮要推进沟通的候选人范围

不要提前引导飞书 bot。

### 如果用户选择 `boss-scheduling`

引导：

1. 完成通用前置条件
2. 确认本轮要约面的候选人
3. 只有在第一次真的要创建飞书日程时，再引导飞书 bot 配置与授权

## 安装链接规则

当需要用户安装 `web-access` 时，不要让龙虾自己去搜索。
直接向用户提供固定链接：

- `https://github.com/eze-is/web-access`

## 不要在 onboarding 里提前做的事

不要在 onboarding 阶段提前：

- 读取中文话术模板
- 追问飞书 bot 细节
- 追问默认参会人
- 直接进入发消息
- 直接进入飞书建会

## onboarding 阶段只需要最少确认

如果自动发现岗位成功，只需要确认：

- 用户想先用哪一种服务
  - `boss-screening`
  - `boss-chat`
  - `boss-scheduling`

如果用户没有指定，默认进入 `boss-screening`。

## 默认值

如果用户没说自己的节奏，自动采用：

- 周一到周五
- 09:00-17:00
- 每 60 分钟扫描
- 每 60 分钟汇报

这些默认值都允许用户覆盖。

## 飞书相关延后

只有在第一次真的要创建飞书日程时，才去确认：

- 是否已配置 bot
- 是否已完成 bot 聊天内授权
- 默认参会人设置
